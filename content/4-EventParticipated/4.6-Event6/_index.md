## Event Report: BUILDING AGENTIC AI - Context Optimization with Amazon Bedrock

### Event Purpose
- The workshop was organized to help participants gain deep understanding of **Agentic AI** and how to build intelligent **AI agents** on **Amazon Bedrock**. The event focused on **context** optimization and building **agentic workflows** in real-world environments. Main objectives:
  - Clearly understand the concept of **Agentic AI** and its role in modern AI systems.
  - Master the architecture and core features of **Amazon Bedrock Agent**.
  - Explore **context optimization** techniques to improve AI agent performance.
  - Learn how to build **agentic workflows** on AWS services.
  - Hands-on practice with **CloudThinker orchestration framework**.

---

- **Viet Pham** – Founder cum CEO (Diaflow)

### Highlighted Content

#### 1. AWS Bedrock Agent Core
**Kien Nguyen** presented the architecture and core capabilities of **Amazon Bedrock**:
- **Agent capabilities**: Key features of Bedrock agents in automated task processing.
- **Core concepts**: 
  - **Foundation Models (FMs)** and agent integration
  - **Agent architecture** and workflow execution
  - **Action groups** and **knowledge bases**
- **Key features**:
  - **Multi-step reasoning** and **task decomposition**
  - **Tool integration** and **API orchestration**
  - **Memory management** and **context handling**
- How to build AI agents capable of automating complex processes on AWS.

#### 2. Building Agentic Workflow on AWS - Real-world Use Case
**Viet Pham** (CEO Diaflow) shared a real-world case study:
- Introduction to **Diaflow** and the business problem to solve.
- **Agentic workflow implementation**:
  - Designing workflows with multiple coordinating agents
  - Integration with AWS services (Lambda, Step Functions, DynamoDB)
  - **Event-driven architecture** for real-time processing
- **Challenges & Solutions**:
  - Handling **context switching** between agents
  - Ensuring **consistency** and **reliability**
  - **Cost optimization** when using Foundation Models
- Results achieved and lessons learned from production deployment.

#### 3. CloudThinker Introduction
**Thang Ton** introduced the **CloudThinker platform**:
- Overview of **CloudThinker** - an **AI orchestration** platform.
- **Product vision**: Simplifying the building and management of AI agents.
- **Core capabilities**:
  - **Agent orchestration** and **workflow management**
  - **Context optimization engine**
  - **Multi-model support** and **model routing**
- CloudThinker's position in the AI/ML ecosystem on AWS.

#### 4. CloudThinker Agentic Orchestration & Context Optimization (L300)
**Henry Bui** presented a technical deep-dive (Level 300):
- **Orchestration Framework**:
  - **Agent coordination patterns** and **communication protocols**
  - **State management** in multi-agent systems
  - **Error handling** and **retry mechanisms**
  
- **Context Optimization Techniques**:
  - **Context window management**: Optimizing token usage
  - **Semantic chunking** and **relevance scoring**
  - **Dynamic context injection**: Only including necessary information in prompts
  - **Context caching strategies** to reduce latency and cost
  
- **Performance Optimization**:
  - **Prompt engineering** best practices
  - **Model selection** based on task complexity
  - **Parallel execution** and **batch processing**
  - **Monitoring & observability** with CloudWatch and X-Ray

- **Integration with Amazon Bedrock**:
  - Efficient use of **Bedrock APIs**
  - **Knowledge base integration** for RAG (Retrieval-Augmented Generation)
  - **Guardrails** and **safety controls**

#### 5. CloudThinker Hack: Hands-on Workshop
**Kha Van** guided the hands-on workshop focusing on two main areas:

- **Connecting to AWS**:
  - Configuring **AWS credentials** and **IAM permissions** for Bedrock
  - Setting up **CloudThinker SDK** and connecting to AWS services
  - Authenticating and testing connection with **Amazon Bedrock APIs**
  - Configuring **security best practices** for production environment
  
- **Cost Optimization**:
  - Analyzing **token usage** and identifying cost drivers
  - Implementing **context caching** to reduce API calls
  - Applying **prompt optimization** to reduce token consumption
  - Using **model selection strategies** - choosing appropriate models for task complexity
  - Monitoring and tracking **cost metrics** with CloudWatch
  - Best practices to **minimize costs** when scaling AI applications
